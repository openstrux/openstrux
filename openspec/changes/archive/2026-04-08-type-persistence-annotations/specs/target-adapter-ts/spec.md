## MODIFIED Requirements

### Requirement: @type record generates a Prisma model and TypeScript interface
For each `@type` record node, the adapter SHALL emit a Prisma `model` block in `prisma/schema.prisma` and a TypeScript `interface` in `types/<RecordName>.ts`. The Prisma model block SHALL include all field-level annotations (`@pk`, `@default`, `@unique`, `@relation`, `@updatedAt`, `@column`, `@ignore`) translated to their Prisma equivalents. External types (`@external`) SHALL NOT produce a `model` block.

#### Scenario: Record produces compilable TypeScript interface
- **WHEN** the adapter processes `@type Proposal { id: string @pk, title: string, status: ReviewStatus }`
- **THEN** `types/Proposal.ts` SHALL contain `export interface Proposal { id: string; title: string; status: ReviewStatus; }`

#### Scenario: Record with @pk produces @id in Prisma model
- **WHEN** the adapter processes `@type Proposal { id: string @pk }`
- **THEN** the Prisma model block SHALL contain `id  String  @id @default(cuid())`

#### Scenario: @pk(default: uuid) produces uuid default
- **WHEN** the adapter processes `@type Proposal { id: string @pk(default: uuid) }`
- **THEN** the Prisma model block SHALL contain `id  String  @id @default(uuid())`

#### Scenario: @pk on int field produces autoincrement default
- **WHEN** the adapter processes `@type Counter { id: number @pk }`
- **THEN** the Prisma model block SHALL contain `id  Int  @id @default(autoincrement())`

#### Scenario: @default(now) produces Prisma @default(now())
- **WHEN** the adapter processes `@type Proposal { createdAt: date @default(now) }`
- **THEN** the Prisma model block SHALL contain `createdAt  DateTime  @default(now())`

#### Scenario: @unique produces Prisma @unique
- **WHEN** the adapter processes `@type User { email: string @unique }`
- **THEN** the Prisma model block SHALL contain `email  String  @unique`

#### Scenario: @relation produces Prisma @relation directive
- **WHEN** the adapter processes `@type Post { author: User @relation(field: authorId, ref: User.id, onDelete: Cascade) }` and `@type User { id: string @pk }`
- **THEN** the `Post` Prisma model SHALL contain `author  User  @relation(fields: [authorId], references: [id], onDelete: Cascade)`
- **AND** the `User` Prisma model SHALL contain an auto-generated inverse field `posts  Post[]`

#### Scenario: @updatedAt produces Prisma @updatedAt
- **WHEN** the adapter processes `@type Post { updatedAt: date @updatedAt }`
- **THEN** the Prisma model block SHALL contain `updatedAt  DateTime  @updatedAt`

#### Scenario: @column produces Prisma @map
- **WHEN** the adapter processes `@type Post { submittedAt: date @column("submitted_at") }`
- **THEN** the Prisma model block SHALL contain `submittedAt  DateTime  @map("submitted_at")`

#### Scenario: @ignore produces Prisma @ignore
- **WHEN** the adapter processes `@type Post { internalNote: string @ignore }`
- **THEN** the Prisma model block SHALL contain `internalNote  String  @ignore`

#### Scenario: @@index produces Prisma @@index
- **WHEN** the adapter processes `@type Proposal { id: string @pk, status: string, @@index([status]) }`
- **THEN** the Prisma model block SHALL contain `@@index([status])`

#### Scenario: @@unique produces Prisma @@unique
- **WHEN** the adapter processes `@type User { firstName: string, lastName: string, @@unique([firstName, lastName]) }`
- **THEN** the Prisma model block SHALL contain `@@unique([firstName, lastName])`

#### Scenario: @@table produces Prisma @@map
- **WHEN** the adapter processes `@type Proposal { @@table("tbl_proposals") }`
- **THEN** the Prisma model block SHALL contain `@@map("tbl_proposals")`

#### Scenario: @timestamps injects createdAt and updatedAt into Prisma model
- **WHEN** the adapter processes `@type Proposal @timestamps { id: string @pk }`
- **THEN** the Prisma model block SHALL contain `createdAt  DateTime  @default(now())` and `updatedAt  DateTime  @updatedAt`

#### Scenario: @external type produces no Prisma model block
- **WHEN** the adapter processes `@external type LegacyUser { id: string, email: string }`
- **THEN** `prisma/schema.prisma` SHALL NOT contain a `model LegacyUser` block

#### Scenario: @opaque content appears as a comment in Prisma model
- **WHEN** the adapter processes `@type Proposal { id: string @pk, @opaque index("idx_status", [status]) }`
- **THEN** the Prisma model block SHALL contain `// @opaque index("idx_status", [status])` as a comment line

#### Scenario: Prisma schema has generated-file header
- **WHEN** the adapter emits `prisma/schema.prisma`
- **THEN** the file SHALL begin with `// generated by openstrux-generator — do not edit`

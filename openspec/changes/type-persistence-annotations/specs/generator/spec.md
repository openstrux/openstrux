## MODIFIED Requirements

### Requirement: Generator output paths are package-relative
All `GeneratedFile.path` values returned by the generator SHALL be relative to the package output directory, not the consumer project root. Exception: `prisma/schema.prisma` is resolved against the `output.prisma_schema` config value and written to the project root by default (see Prisma schema output path requirement below).

#### Scenario: Type file path is package-relative
- **WHEN** the generator emits a type file for `@type Proposal`
- **THEN** the `GeneratedFile.path` SHALL be `types/Proposal.ts`, not `src/types/Proposal.ts` or an absolute path

#### Scenario: Prisma schema path is project-root-relative
- **WHEN** the generator emits the Prisma schema file
- **THEN** the file SHALL be written to the path specified by `output.prisma_schema` in `strux.config.yaml`, defaulting to `prisma/schema.prisma` at the project root

## ADDED Requirements

### Requirement: Prisma schema output path is configurable and defaults to project root
The `strux build` command SHALL write `prisma/schema.prisma` to the project root by default. The output path SHALL be configurable via `output.prisma_schema` in `strux.config.yaml`.

#### Scenario: Default output path is project root
- **WHEN** `strux.config.yaml` contains no `output.prisma_schema` key
- **THEN** the Prisma schema SHALL be written to `prisma/schema.prisma` relative to the project root

#### Scenario: Custom output path is respected
- **WHEN** `strux.config.yaml` contains `output.prisma_schema: "packages/db/prisma/schema.prisma"`
- **THEN** the Prisma schema SHALL be written to that path relative to the project root

---

### Requirement: Generator guards against overwriting hand-edited Prisma schemas
Before writing `prisma/schema.prisma`, `strux build` SHALL check whether the file already exists and begins with the generated-file header (`// generated by openstrux-generator — do not edit`). If the file exists but lacks this header, the build SHALL print a warning, skip the write, and exit with a non-zero status unless `--overwrite-schema` is passed.

#### Scenario: Missing header triggers warning and skip
- **WHEN** `prisma/schema.prisma` exists and does not start with `// generated by openstrux-generator — do not edit`
- **THEN** `strux build` SHALL print a warning identifying the file path and SHALL NOT overwrite it, and SHALL exit with status 1

#### Scenario: --overwrite-schema bypasses the guard
- **WHEN** `strux build --overwrite-schema` is invoked and the file exists without the header
- **THEN** the build SHALL overwrite the file with the generated schema

#### Scenario: File with correct header is always overwritten
- **WHEN** `prisma/schema.prisma` exists and starts with the generated header
- **THEN** `strux build` SHALL overwrite it unconditionally (desired-state semantics)

#### Scenario: Non-existent file is always written
- **WHEN** `prisma/schema.prisma` does not exist
- **THEN** `strux build` SHALL create it regardless of `--overwrite-schema`

here is a policy-safe draft you can reuse for an NLnet application. It is designed to align with NLnet’s requirement for disclosure of GenAI use, its ban on presenting AI-generated work as human-authored work, and its explicit support for deterministic, specification-based automation.

Positioning text
You can use this as a proposal paragraph:

Openstrux is a specification-driven software building system for generating reproducible implementation artifacts from explicit structured models, typed constraints, and verifiable composition rules. The project is not about replacing human engineering with generative AI, but about improving software quality, auditability, portability, and reproducibility through machine-readable structure and deterministic generation. Human contributors remain responsible for the architecture, the specifications, the generated outputs, the verification process, and all integration decisions. Where automation is used, the primary goal is to derive auditable outputs from declared specifications rather than to submit opaque AI-produced work. This fits NLnet’s support for automation and deterministic generation based on specifications and engineering rigour, while respecting NLnet’s requirement that grantees do not present AI-generated content as if it were their own human-authored work.
​

GenAI disclosure
If you used GenAI to help prepare the proposal, you can use this short disclosure statement:

Generative AI tools were used in a limited supporting role during proposal preparation, for example for drafting assistance, wording refinement, summarisation, or translation support. Final content selection, technical claims, scope definition, and submission responsibility remain fully human. A prompt provenance log has been maintained, including model used, dates and times, prompts, and unedited outputs, in line with NLnet’s application policy.
​

If you used no GenAI in the proposal, use this instead:

No generative AI tools were used to draft or prepare this proposal.
​

Project policy note
If Openstrux itself may involve some GenAI-assisted development, add a project-policy paragraph like this:

This project may experiment with generative AI tools in a transparent and limited manner for selected support tasks, but all substantive project outputs will remain under human responsibility for correctness, originality, clarity, and reproducibility. Any substantive GenAI use affecting project outputs will be publicly disclosed in project documentation, and generated content will be marked with provenance information such as model/version and prompt or interaction records, or an equivalent log. Purely AI-generated outputs will not be submitted as grant work eligible for payment.
​

Recommended stance
The safest stance for NLnet is to say that Openstrux primarily delivers deterministic generation from specifications, and that any GenAI use is optional, disclosed, and non-authoritative. That keeps you close to NLnet’s policy, which explicitly distinguishes GenAI from acceptable deterministic automation and reproducible generation based on specifications.

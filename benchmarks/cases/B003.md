# B003 — Union Type

**id:** B003
**category:** token-efficiency
**constructs:** `@type` union (discriminated)

## Intent Prompt

Define a `ReviewDecision` union with 3 variants: `approved` carrying an `approvedAmount` (number), `rejected` carrying a `reason` (string), and `deferred` carrying a `reviewDate` (date). Model a grant review outcome.

## Openstrux Source

```
@type ApprovedPayload  { approvedAmount: number }
@type RejectedPayload  { reason: string }
@type DeferredPayload  { reviewDate: date }

@type ReviewDecision = union {
  approved: ApprovedPayload,
  rejected: RejectedPayload,
  deferred: DeferredPayload
}
```

## TypeScript Baseline

```typescript
// TypeScript discriminated union
export type ReviewDecision =
  | { kind: "approved"; approvedAmount: number }
  | { kind: "rejected"; reason: string }
  | { kind: "deferred"; reviewDate: Date };
```

_(No direct Prisma equivalent — typically stored as a JSON column or separate tables. Minimal hand-written mapping shown.)_

```typescript
// Prisma storage helper (representative minimal mapping)
export function decisionToJson(d: ReviewDecision): Record<string, unknown> {
  return { kind: d.kind, ...d };
}
export function decisionFromJson(raw: Record<string, unknown>): ReviewDecision {
  if (raw.kind === "approved") return { kind: "approved", approvedAmount: raw.approvedAmount as number };
  if (raw.kind === "rejected") return { kind: "rejected", reason: raw.reason as string };
  return { kind: "deferred", reviewDate: new Date(raw.reviewDate as string) };
}
```
